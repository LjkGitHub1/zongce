# 综测材料收集系统数据库设计文档

## 1. 数据库概述

- **数据库类型**：MySQL 8.0
- **字符集**：utf8mb4
- **排序规则**：utf8mb4_unicode_ci
- **存储引擎**：InnoDB

## 2. 表结构设计

### 2.1 用户表（user）

| 字段名 | 类型 | 长度 | 是否为空 | 主键 | 说明 |
|--------|------|------|----------|------|------|
| id | BIGINT | - | NOT NULL | PK, AUTO_INCREMENT | 主键ID |
| student_id | VARCHAR | 50 | NOT NULL | UNIQUE | 学号（唯一） |
| name | VARCHAR | 100 | NOT NULL | - | 姓名 |
| identity | VARCHAR | 20 | NOT NULL | - | 身份（teacher/graduate/undergraduate） |
| major | VARCHAR | 100 | NULL | - | 专业 |
| class | VARCHAR | 50 | NULL | - | 班级 |
| password | VARCHAR | 255 | NOT NULL | - | 密码（加密存储） |
| email | VARCHAR | 100 | NULL | - | 邮箱 |
| phone | VARCHAR | 20 | NULL | - | 手机号 |
| is_active | BOOLEAN | - | NOT NULL | - | 是否激活（默认True） |
| is_staff | BOOLEAN | - | NOT NULL | - | 是否管理员（默认False） |
| created_at | DATETIME | - | NOT NULL | - | 创建时间 |
| updated_at | DATETIME | - | NOT NULL | - | 更新时间 |

**索引：**
- PRIMARY KEY (id)
- UNIQUE KEY (student_id)
- INDEX idx_identity (identity)
- INDEX idx_major (major)
- INDEX idx_class (class)

## 3. 表关系说明

### 3.1 外键关系
- `user.student_id` → 所有材料表的 `student_id`（逻辑外键）
- `file.upload_user_id` → `user.id`
- `file.material_id` → 各材料表的 `id`（逻辑外键）
- `audit_record.auditor_id` → `user.id`
- `audit_record.material_id` → 各材料表的 `id`（逻辑外键）
- `announcement.publisher_id` → `user.id`
- `message.user_id` → `user.id`
- `operation_log.user_id` → `user.id`

### 3.2 关系说明
- **用户 ↔ 材料表**：一对多关系（一个用户可以有多个材料）
- **材料 ↔ 文件表**：一对多关系（一个材料可以有多个证明文件）
- **材料 ↔ 审核记录**：一对多关系（一个材料可以有多次审核记录）
- **用户 ↔ 消息**：一对多关系（一个用户可以有多条消息）

## 4. 状态枚举值

### 4.1 材料状态（status）
- `pending`：待审核
- `auditing`：审核中
- `approved`：已通过
- `rejected`：已驳回
- `withdrawn`：已撤回

### 4.2 用户身份（identity）
- `teacher`：老师（管理员）
- `graduate`：研究生
- `undergraduate`：本科生

### 4.3 审核结果（result）
- `approved`：通过
- `rejected`：驳回
