# 综测材料收集系统需求文档

## 1. 项目概述

### 1.1 项目背景
综测材料收集系统是一个用于收集、管理和审核学生综合素质测评相关材料的在线平台。系统方便老师和学生上传、下载各类证明材料，管理员进行审核和统计，最终形成学生的综测成绩依据。

### 1.2 项目目标
- 实现学生材料的在线提交、管理和下载
- 提供管理员审核功能
- 支持材料分类统计和数据导出
- 提高材料收集效率，减少人工操作
- 确保数据的准确性和可追溯性

### 1.3 用户角色与权限
- **管理员（老师）**：拥有所有权限
  - 审核所有材料
  - 管理用户信息
  - 查看统计数据
  - 系统配置（权限配置、公告管理等）
  
- **研究生**：需提交所有材料模块
  - 学术成果模块（课题项目、专著、论文、其他学术成果）
  - 服务与实践模块（科技竞赛、社会实践调研、服务社会、其他实践活动）
  
- **本科生**：目前只需提交科技竞赛模块（权限可配置，后期可能扩展）

## 2. 核心功能需求

### 2.1 用户管理

#### 2.1.1 用户认证
- 用户注册（学号、姓名、身份：老师/研究生/本科生、专业、班级）
- 用户登录（JWT认证）
- 密码找回
- 管理员账号系统初始化创建

#### 2.1.2 用户信息
- 基础信息：学号（student_id）、姓名（name）、身份（identity）、专业（major）、班级（class）
- 用户查看/修改个人信息
- 管理员批量导入用户（Excel）
- 权限配置管理（可配置不同身份用户可访问的模块）

### 2.2 学术成果模块（academic_achievements）

#### 2.2.1 课题项目（research_project）
**字段要求：**
- 项目名称（project_name）
- 项目级别（level）
- 立项时间（start_date）
- 结题时间（end_date）
- 项目经费（万元）（funds）
- 项目编号（number）
- 主持人、参与人员及顺序（personnel）
- 证明材料（proof）- 需上传电子版
- 备注（remark）

#### 2.2.2 专著（monograph）
**字段要求：**
- 专著名称（title）
- 出版社（publish）
- 出版时间（publish_date）
- 编号（number）
- 导师姓名（supervisor）
- 撰写字数（word_count）
- 作者及顺序（author）
- 证明材料（proof）- 需上传电子版
- 备注（remark）

#### 2.2.3 论文（paper）
**字段要求：**
- 论文题目（title）
- 刊物名称（publication）
- 类别（category）
- 刊号（publication_number）
- 发表时间（publication_date）
- 期刊收录情况（journal_status）
- 中科院分区（partition）
- 导师姓名（supervisor）
- 是否第一单位（first_unit）
- 导师是否通讯（communication）
- 作者及排名（author）
- 证明材料（proof）- 需上传电子版
- 备注（remark）

#### 2.2.4 其他学术成果（other_academic）
**字段要求：**
- 富文本内容（请注明成果名称、时间等详细信息）
- 证明材料（proof）

### 2.3 服务与实践模块（service_practice）

#### 2.3.1 科技竞赛（technology_competition）
**字段要求：**
- 赛事名称（competition）
- 是否为白名单赛事（是/否）
- 作品名称（title）
- 级别（level）：国家级/省级/校级/院级（下拉选择）
- 获奖时间（award_date）- 格式：XX年XX月
- 获奖等级（grade）
- 组织单位名称（organization）
- 团体/个人（group）
- 负责人/参与者及顺序（author）
- 指导老师（supervisor）
- 证明材料（proof）- 需上传电子版
- 备注（remark）

#### 2.3.2 社会实践调研（social_practice）
**字段要求：**
- 名称（title）
- 级别（level）
- 时间（date）
- 主持人、参与者、顺序（author）
- 证明材料（proof）- 需上传电子版
- 备注（remark）

#### 2.3.3 服务社会（social_service）
**字段要求：**
- 名称（title）
- 级别（level）
- 时间（date）
- 作者及顺序（author）
- 证明材料（proof）- 需上传电子版
- 备注（remark）

#### 2.3.4 其他实践活动（other_practice）
**字段要求：**
- 名称（title）
- 级别（level）
- 时间（date）
- 提供证明单位（organization）
- 参与人（author）
- 证明材料（proof）- 需上传电子版
- 备注（remark）

### 2.4 材料管理

#### 2.4.1 材料提交
- 表单填写（各模块对应字段）
- 文件上传（支持图片：JPG、PNG；文档：PDF、DOC、DOCX）
- 单文件/批量上传
- 文件大小限制（单文件≤10MB）
- 上传进度显示
- 使用Celery异步处理大文件

#### 2.4.2 材料状态
- **待审核**：刚提交的材料
- **审核中**：管理员正在审核
- **已通过**：审核通过
- **已驳回**：审核未通过（需填写驳回原因）
- **已撤回**：用户主动撤回

#### 2.4.3 材料查看与管理
**用户端：**
- 查看个人所有材料
- 按状态/类型/时间筛选排序
- 编辑/删除未审核材料
- 查看审核意见

**管理员端：**
- 查看所有用户材料
- 多条件筛选（用户、类型、状态、时间、身份、专业、班级）
- 材料详情查看
- 批量操作
- 材料导出（Excel）

### 2.5 审核管理

#### 2.5.1 审核功能
- 待审核材料列表
- 查看材料详情和附件
- 审核通过/驳回
- 填写审核意见（驳回必填）
- 批量审核
- 审核记录（审核人、时间、结果、意见）

#### 2.5.2 审核统计
- 待审核数量
- 审核通过率
- 按时间段统计

### 2.6 数据统计

#### 2.6.1 用户统计
- 个人材料总数
- 按类型/状态统计
- 个人综测得分（基于已通过材料）

#### 2.6.2 管理员统计
- 全校/全院材料总数
- 按类型/时间段/身份/专业/班级统计
- 审核情况统计
- 用户参与度统计
- 数据可视化（图表）

### 2.7 通知公告

#### 2.7.1 公告管理
- 管理员发布公告
- 公告分类（系统通知、审核通知、重要通知）
- 公告置顶
- 公告有效期

#### 2.7.2 消息通知
- 审核结果通知（通过/驳回）
- 系统消息推送
- 个人消息列表

### 2.8 数据导出

- 用户材料清单（Excel）
- 审核统计报表
- 综测成绩汇总表
- 自定义导出字段
- 按条件筛选导出（支持按身份、专业、班级等筛选）
- 按身份/专业/班级分类导出

## 3. 数据库设计

### 3.1 基础字段
每个模型都包含：
- **student_id**：学号
- **name**：姓名
- **identity**：身份（老师/研究生/本科生）

### 3.2 核心数据表

#### 3.2.1 用户表（User）
- 用户基本信息（学号、姓名、身份、专业、班级、密码等）
- 权限配置

#### 3.2.2 学术成果表
- 课题项目表（ResearchProject）
- 专著表（Monograph）
- 论文表（Paper）
- 其他学术成果表（OtherAcademic）

#### 3.2.3 服务与实践表
- 科技竞赛表（TechnologyCompetition）
- 社会实践调研表（SocialPractice）
- 服务社会表（SocialService）
- 其他实践活动表（OtherPractice）

#### 3.2.4 系统管理表
- 审核记录表（审核人、时间、结果、意见）
- 公告表
- 消息表
- 操作日志表
- 权限配置表

### 3.3 数据关系
- 用户 ↔ 各类材料表：一对多（通过student_id）
- 材料 ↔ 证明材料：一对多
- 材料 ↔ 审核记录：一对多
- 用户 ↔ 消息：一对多

## 4. 技术架构

### 4.1 前端技术栈
- **框架**：Vue 3
- **UI组件库**：Element Plus
- **高级组件库**：Plus Pro Components
- **构建工具**：Vite
- **状态管理**：Pinia
- **路由**：Vue Router

### 4.2 后端技术栈
- **框架**：Django 5.0
- **API框架**：Django REST Framework
- **异步任务**：Celery
- **过滤**：django-filter
- **认证**：django-rest-framework-simplejwt（JWT）
- **API设计**：RESTful API

### 4.3 数据库
- **数据库**：MySQL 8.0
- **ORM**：Django ORM

### 4.4 其他技术
- **缓存/消息队列**：Redis（Celery消息队列和缓存）
- **Web服务器**：Gunicorn + Nginx（生产环境）
- **部署**：Docker容器化部署

## 5. 非功能需求

### 5.1 性能需求
- 页面加载时间 ≤ 3秒
- 支持至少100人并发
- 文件上传支持断点续传

### 5.2 安全需求
- 密码加密存储
- 文件上传安全验证（类型、大小）
- 防止SQL注入、XSS攻击
- 操作日志记录
- 数据备份机制

### 5.3 可用性需求
- 界面简洁易用
- 支持主流浏览器（Chrome、Firefox、Edge、Safari）
- 响应式设计，支持移动端
- 操作提示清晰，错误信息明确
