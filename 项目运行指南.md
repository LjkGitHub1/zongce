## 环境要求

### 必需软件

1. **Python 3.8+**
   - 下载地址：https://www.python.org/downloads/

3. **MySQL 8.0**
   - 下载地址：https://dev.mysql.com/downloads/mysql/

4. **Git**（可选）
   - 下载地址：https://git-scm.com/downloads

### 推荐工具

- **代码编辑器**：VS Code、PyCharm
- **数据库管理工具**：Navicat、phpMyAdmin、MySQL Workbench
- **API测试工具**：Postman（可选）

## 后端部署

### 步骤 1：进入后端目录

```bash
cd backend
```

### 步骤 2：创建虚拟环境（推荐）

**Windows系统：**
```bash
python -m venv venv
venv\Scripts\activate
```

**Linux/Mac系统：**
```bash
python3 -m venv venv
source venv/bin/activate
```

> **提示**：激活虚拟环境后，命令行前面会显示 `(venv)`

### 步骤 3：安装Python依赖

```bash
pip install -r requirements.txt
```

如果安装速度慢，可以使用国内镜像：
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

**常见问题**：
- 如果 `mysqlclient` 安装失败，请先安装 MySQL 开发库
  - Windows：下载 MySQL Connector/C 或使用预编译的 wheel 文件
  - Linux：`sudo apt-get install python3-dev default-libmysqlclient-dev`
  - Mac：`brew install mysql pkg-config`

### 步骤 4：配置环境变量

在 `backend` 目录下创建 `.env` 文件（或设置系统环境变量）：

```env
# 数据库配置
DB_NAME=zongce
DB_USER=root
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=3306

# Django密钥（生产环境请修改）
SECRET_KEY=your-secret-key-here

# Celery配置（可选，如果使用Redis）
CELERY_BROKER_URL=redis://localhost:6379/0
CELERY_RESULT_BACKEND=redis://localhost:6379/0
```

> **注意**：`.env` 文件不要提交到版本控制系统

### 步骤 5：创建数据库

使用 MySQL 客户端或命令行创建数据库：

```sql
CREATE DATABASE zongcesys CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

或者使用命令行：
```bash
mysql -u root -p
```

然后执行：
```sql
CREATE DATABASE zongcesys CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
```

### 步骤 6：运行数据库迁移

```bash
python manage.py makemigrations
python manage.py migrate
```

### 步骤 7：创建超级用户（管理员）

```bash
python manage.py createsuperuser
```

按提示输入：
- 学号（作为用户名）
- 姓名
- 身份（输入 `teacher`）
- 邮箱（可选）
- 密码（输入两次）

### 步骤 8：收集静态文件（可选）

```bash
python manage.py collectstatic
```

## 前端部署

### 步骤 1：进入前端目录

```bash
cd frontend
```

### 步骤 2：安装依赖

```bash
npm install
```

如果安装速度慢，可以使用国内镜像：
```bash
npm install --registry=https://registry.npmmirror.com
```

或者设置npm镜像：
```bash
npm config set registry https://registry.npmmirror.com
npm install
```

### 步骤 3：配置API地址（如需要）

编辑 `frontend/vite.config.js`，确认代理配置：

```javascript
server: {
  port: 5173,
  proxy: {
    '/api': {
      target: 'http://localhost:8000',  // 后端地址
      changeOrigin: true
    }
  }
}
```

## 数据库配置

### MySQL 配置检查

1. **确认MySQL服务运行**
   - Windows：服务管理器中查看 MySQL 服务
   - Linux：`sudo systemctl status mysql`
   - Mac：系统偏好设置中查看

2. **测试数据库连接**

创建测试脚本 `test_db.py`：
```python
import mysql.connector

try:
    conn = mysql.connector.connect(
        host='localhost',
        user='root',
        password='your_password',
        database='zongce'
    )
    print("数据库连接成功！")
    conn.close()
except Exception as e:
    print(f"数据库连接失败：{e}")
```

运行测试：
```bash
python test_db.py
```

### 数据库表结构

运行迁移后，系统会自动创建以下表：
- `user` - 用户表
- `research_project` - 课题项目表
- `monograph` - 专著表
- `paper` - 论文表
- `technology_competition` - 科技竞赛表
- `file` - 文件表
- `audit_record` - 审核记录表
- `announcement` - 公告表
- `message` - 消息表
- 等等...

## 启动项目

### 启动后端

1. **确保虚拟环境已激活**
   ```bash
   # Windows
   venv\Scripts\activate
   
   # Linux/Mac
   source venv/bin/activate
   ```

2. **进入后端目录**
   ```bash
   cd backend
   ```

3. **启动Django开发服务器**
   ```bash
   python manage.py runserver
   ```

   看到以下输出表示启动成功：
   ```
   Starting development server at http://127.0.0.1:8000/
   Quit the server with CTRL-BREAK.
   ```

4. **测试后端API**
   - 打开浏览器访问：http://127.0.0.1:8000/admin/
   - 使用超级用户账号登录
   - 或使用Postman测试API：http://127.0.0.1:8000/api/auth/login/

### 启动前端

1. **打开新的终端窗口**（保持后端运行）

2. **进入前端目录**
   ```bash
   cd frontend
   ```

3. **启动开发服务器**
   ```bash
   npm run dev
   ```

   看到以下输出表示启动成功：
   ```
   VITE v5.x.x  ready in xxx ms

   ➜  Local:   http://localhost:5173/
   ➜  Network: use --host to expose
   ```

4. **访问前端应用**
   - 打开浏览器访问：http://localhost:5173/
   - 使用注册的账号登录

### 启动Redis（可选，用于Celery）

如果使用异步任务功能，需要启动Redis：

**Windows：**
- 下载Redis for Windows
- 运行 `redis-server.exe`

**Linux：**
```bash
sudo systemctl start redis
```

**Mac：**
```bash
brew services start redis
```

## 常见问题

### Q1: `pip install` 安装失败

**解决方案**：
- 升级pip：`python -m pip install --upgrade pip`
- 使用国内镜像：`pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`
- 检查网络连接

### Q2: `mysqlclient` 安装失败

**Windows解决方案**：
1. 下载预编译的wheel文件：https://www.lfd.uci.edu/~gohlke/pythonlibs/#mysqlclient
2. 安装：`pip install mysqlclient‑x.x.x‑cp3x‑cp3x‑win_amd64.whl`
3. 或使用 `pymysql` 替代（需要修改settings.py）

**Linux解决方案**：
```bash
sudo apt-get install python3-dev default-libmysqlclient-dev
pip install mysqlclient
```

**Mac解决方案**：
```bash
brew install mysql pkg-config
pip install mysqlclient
```

### Q3: 数据库连接失败

**检查项**：
1. MySQL服务是否运行
2. 数据库是否创建
3. 用户名密码是否正确
4. 端口是否正确（默认3306）
5. 防火墙是否阻止连接

**测试连接**：
```bash
mysql -u root -p -h localhost
```

### Q4: `npm install` 失败或很慢

**解决方案**：
```bash
# 使用国内镜像
npm config set registry https://registry.npmmirror.com
npm install

# 或使用yarn
npm install -g yarn
yarn install
```

### Q5: 前端无法连接后端API

**检查项**：
1. 后端是否启动（http://127.0.0.1:8000）
2. `vite.config.js` 中的代理配置是否正确
3. 浏览器控制台是否有CORS错误
4. 检查后端 `settings.py` 中的 `CORS_ALLOWED_ORIGINS`

### Q6: 迁移失败

**解决方案**：
```bash
# 删除所有迁移文件（除了__init__.py）
# 重新生成迁移
python manage.py makemigrations
python manage.py migrate

# 如果仍有问题，删除数据库重新创建
```

### Q7: 端口被占用

**后端端口（8000）被占用**：
```bash
# Windows查看端口占用
netstat -ano | findstr :8000
# 结束进程
taskkill /PID <进程ID> /F

# Linux/Mac查看端口占用
lsof -i :8000
# 结束进程
kill -9 <进程ID>

# 或使用其他端口
python manage.py runserver 8001
```

**前端端口（5173）被占用**：
```bash
# 修改 vite.config.js
server: {
  port: 5174,  // 改为其他端口
}
```

### Q8: 静态文件404错误

**解决方案**：
```bash
python manage.py collectstatic
```

### Q9: 文件上传失败

**检查项**：
1. `media` 目录是否存在且有写权限
2. 文件大小是否超过限制（10MB）
3. 文件格式是否支持

### Q10: 无法创建超级用户

**解决方案**：
```bash
# 使用Django shell创建
python manage.py shell
```

```python
from apps.users.models import User
User.objects.create_superuser(
    student_id='admin001',
    password='your_password',
    name='管理员',
    identity='teacher'
)
```

## 生产环境部署（简要说明）

### 后端部署

1. **使用Gunicorn**
   ```bash
   pip install gunicorn
   gunicorn config.wsgi:application --bind 0.0.0.0:8000
   ```

2. **使用Nginx反向代理**
   - 配置Nginx指向Gunicorn
   - 配置静态文件和媒体文件

3. **使用Docker（推荐）**
   - 创建Dockerfile
   - 使用docker-compose编排服务

### 前端部署

1. **构建生产版本**
   ```bash
   npm run build
   ```

2. **部署到服务器**
   - 将 `dist` 目录部署到Nginx
   - 或使用CDN
